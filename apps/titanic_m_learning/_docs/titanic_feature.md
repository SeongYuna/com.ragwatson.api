# 타이타닉 데이터셋 설명 (피처 엔지니어링 참고용)

## 개요

데이터는 두 그룹으로 나뉩니다:

- **훈련 세트** (`train.csv`): 머신러닝 모델 학습에 사용. 각 승객의 생존 여부(정답 레이블) 포함.
- **테스트 세트** (`test.csv`): 모델 성능 평가에 사용. 생존 여부 미제공 → 예측 대상.
- **참고 파일** (`gender_submission.csv`): "여성 승객만 생존"을 가정한 예시 제출 파일.

---

## 데이터 사전 (변수 설명)

| 변수명 | 설명 | 값/키 |
|--------|------|-------|
| `survival` | 생존 여부 (타겟 변수) | 0 = 사망, 1 = 생존 |
| `pclass` | 티켓 등급 (사회경제적 지위 대리 변수) | 1 = 1등석(상류층), 2 = 2등석(중산층), 3 = 3등석(하류층) |
| `sex` | 성별 | male / female |
| `age` | 나이 (연 단위) | 1세 미만은 소수점 표기; 추정 나이는 `xx.5` 형식 |
| `sibsp` | 함께 탑승한 형제자매 / 배우자 수 | 형제·자매·이복형제·이복자매 / 남편·아내 포함 (내연관계·약혼자 제외) |
| `parch` | 함께 탑승한 부모 / 자녀 수 | 부모(어머니·아버지) / 자녀(딸·아들·의붓딸·의붓아들) 포함; 유모와만 동행한 어린이는 0 |
| `ticket` | 티켓 번호 | 문자+숫자 혼합 가능 |
| `fare` | 운임 요금 | 연속형 수치 |
| `cabin` | 객실 번호 | 결측값 다수; 알파벳(덱 정보) + 숫자 구조 |
| `embarked` | 탑승 항구 | C = 셰르부르(Cherbourg), Q = 퀸스타운(Queenstown), S = 사우샘프턴(Southampton) |

## 피처 엔지니어링 힌트

### `pclass` (티켓 등급)
- 1등석일수록 생존율이 높은 경향 → 생존과 강한 상관관계
- 범주형으로 처리하거나 서수형(ordinal)으로 활용 가능

### `sex` (성별)
- "여성과 어린이 우선" 원칙 → 생존에 가장 강력한 변수 중 하나
- 이진 인코딩(male=0, female=1) 적용

### `age` (나이)
- 결측값 다수 → 중앙값, 그룹별 중앙값, 또는 모델 기반 대체(imputation) 권장
- 나이 구간화(binning): 어린이(0~12), 청소년(13~17), 성인(18~60), 노인(61+) 등으로 그룹화 고려
- `age * pclass` 등 교호작용 피처 생성 가능

### `sibsp` + `parch` (가족 관계)
- **파생 피처 제안**:
  - `family_size = sibsp + parch + 1` (본인 포함 탑승 가족 수)
  - `is_alone = 1 if family_size == 1 else 0` (단독 탑승 여부)
- 혼자 탑승한 승객과 소규모 가족의 생존율 차이 분석 권장
- 대가족(family_size > 4)은 오히려 생존율 낮을 수 있음

### `ticket` (티켓 번호)
- **파생 피처 제안**:
  - 티켓 접두사(prefix) 추출: 그룹이나 선실 위치 정보 내포 가능
  - 같은 티켓 번호를 공유하는 승객 수 → 동행 그룹 크기 파악 가능

### `fare` (운임 요금)
- 결측값 소수 → 중앙값 대체
- 로그 변환(`log1p`) 으로 왜도(skewness) 완화 권장
- `pclass`와 상관성 높음 → 다중공선성 고려

### `cabin` (객실 번호)
- 결측값 매우 많음(~77%)
- **파생 피처 제안**:
  - `has_cabin = 1 if cabin is not null else 0` (객실 정보 유무)
  - 첫 알파벳 추출 → 덱(Deck) 정보: A, B, C, D, E, F, G, T
  - 덱별 생존율 차이 분석 가능

### `embarked` (탑승 항구)
- 결측값 2개 → 최빈값(S) 대체
- 원-핫 인코딩(One-Hot Encoding) 또는 레이블 인코딩 적용
- 항구별 `pclass` 분포 차이 존재 → 간접적으로 사회경제적 정보 반영

### 추가 파생 피처 아이디어
- `title` (호칭): `name` 컬럼에서 Mr., Mrs., Miss., Master. 등 추출 → 나이·성별·지위 대리 변수
- `fare_per_person = fare / family_size`: 실질 지불 운임 추정
- `age_class = age * pclass`: 나이와 등급의 교호작용

## 결측값 현황 요약

| 변수 | 결측 정도 | 권장 대체 방법 |
|------|----------|----------------|
| `age` | 중간 (~20%) | 그룹별 중앙값 또는 모델 기반 대체 |
| `cabin` | 매우 많음 (~77%) | 유무 이진화 후 덱 추출 |
| `embarked` | 거의 없음 (2개) | 최빈값(S) 대체 |
| `fare` | 거의 없음 (1개, 테스트셋) | 중앙값 대체 |
